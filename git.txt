1. 사용자 설정하기

사용자 이름 설정하기
git config --global user.name "이름작성"

사용자 이름 확인하기
git config --global user.name

사용자 이메일 설정하기
git config --global user.email "이메일작성"

사용자 이메일 확인하기
git config --global user.email

2. 폴더를 저장소로 초기화 하기
git init
-> 폴더에 .git 폴더가 생성됨

3. git 저장소 폴더에 파일 생성
01.txt 작성후 내용 입력

4. git 저장소 상태 확인
git status
-> 01.txt 파일이 관리되고 있지 않습니다!!

5. 01.txt 파일을 관리 대상에 등록하기
git add 01.txt
-> git status -> 관리 대상에 새파일 01.txt가 있습니다!

6. 관리 대상 파일들의 변경 내용을, 형상관리해줘
git commit -m "파일추가 : 01.txt"
-> 형상관리 내역에 01.txt 파일 추가한 내용과, 01.txt의 내용에 대해서 기록

7. 관리되고 있는 파일의 내용을 변경
01.txt에 내용 추가
-> git add 01.txt
-> git commit -m "수정 : 01.txt"
-> git statu 로 상태 확인

8. 형상관리 이력 확인하기
git log
* 한줄씩 깔끔하게 보기
git log --oneline

9. 이전 커밋으로 되돌리기
git reset --soft 커밋해쉬값
-> checkout -> git 2.23 버전부터 명령 변경
--soft -> 되돌리기 이전의 내용을 staged 상태로 남김

** git reset --hard 커밋해쉬값 **
선택된 커밋 이후의 모든 작업 완전 삭제 : 복구 불가

10. staged 취소하기
git restore --staged 01.txt

11. 파일 수정 내용 취소하기
git restore 01.txt

12. .gitignore 파일 작성 및 활용
.gitignore 파일 생성

.gitignore 파일에 형상관리 하지 않을 파일명, 폴더명 작성

13. .gitignore 에 작성된 파일, 폴더는 add에 잡히지 않습니다
